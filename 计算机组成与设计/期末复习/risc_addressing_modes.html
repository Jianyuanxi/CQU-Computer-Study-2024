RISC-V 五种寻址方式交互讲解

五种寻址方式
点击下方标签，逐一了解每种寻址方式
R 型指令
1. 寄存器寻址
操作数全部来自寄存器，结果也写回寄存器。三个字段分别指定两个源寄存器和一个目标寄存器。
指令格式（32位）
opcode7位
rd5位
funct33位
rs15位
rs25位
funct77位
寻址原理： 指令中的 rs1、rs2 字段直接是寄存器编号（0~31）。CPU 从寄存器堆中读出两个数，运算后存入 rd 寄存器。操作数就在 CPU 内部，速度最快。
add x1, x2, x3 # x1 = x2 + x3（完全在寄存器中）
32个通用寄存器
RISC-V 有 x0~x31 共 32 个寄存器，每个 64 位（RV64）
x0=0
x1=ra
x2=sp
…
x0 恒为 0，无法修改；x1 通常存返回地址；x2 是栈指针
I 型指令
2. 立即数寻址
操作数之一直接编码在指令中，称为"立即数"。无需访问内存或额外寄存器，适合常数运算。
指令格式（32位）
opcode7位
rd5位
funct33位
rs15位
imm[11:0] 立即数12位
寻址原理： 指令本身就携带了一个 12 位常数（符号扩展到 64 位）。CPU 直接使用这个数字作为操作数，不用去取内存或其他寄存器。
addi x1, x2, 100 # x1 = x2 + 100（100就写在指令里）
直观理解
指令中的立即数
100
→
直接用于 ALU
+100
12位能表示 -2048 ~ +2047 的整数，常用于循环计数、地址偏移等
I 型指令
3. 基址偏移寻址（Base + Offset）
用于访问内存。实际内存地址 = 基址寄存器的值 + 指令中的偏移量（立即数）。适合访问数组、结构体字段。
指令格式（同 I 型）
opcode7位
rd5位
funct33位
base5位
offset 偏移量12位
寻址原理： rs1（基址寄存器）+ imm（偏移量）= 内存地址，再从该地址读取数据。
ld x1, 8(x2) # 读取地址 (x2 + 8) 处的数据到 x1
内存访问示意
假设 x2 = 1000，执行 ld x1, 8(x2)
地址 1000
← x2（基址）
地址 1004
地址 1008
← 实际访问（1000+8）✓
地址 1012
常用于读写数组：x2指向数组开头，偏移量选择第几个元素
I / B / S 型指令
4. PC相对寻址
目标地址 = 当前 PC（程序计数器）+ 符号扩展的偏移量。用于条件分支跳转，跳转范围是以当前指令为中心的±一定范围。
核心公式
目标地址
=
PC
+
offset×2
为什么用PC相对？ 程序加载到内存的位置可能不固定，但跳转指令和目标往往在附近。用相对偏移，代码搬到哪里都能正确跳转（位置无关代码）。
beq x1, x2, 16 # 若x1==x2，跳转到 PC+16 的位置

（不跳时下一条）
addr 1012
← 跳转目标（1004+8）✓
J 型指令
5. 伪直接寻址（Pseudo-direct）
用于无条件跳转（JAL）。目标地址大部分位来自指令中的立即数，高位来自当前 PC，因此称"伪直接"——看起来像直接给地址，实则仍有PC的参与。
J 型指令格式（32位）
opcode7位
rd5位
imm[20|10:1|11|19:12] 目标偏移20位
寻址原理： 目标地址 = PC + 符号扩展(imm×2)，imm 有 20 位，跳转范围最远可达 ±1MB。同时将 PC+4 存入 rd（返回地址），用于函数调用。
jal x1, func # 跳到func，返回地址存x1（ra寄存器）
与PC相对寻址的区别
PC相对
用于条件分支（beq/bne），偏移量12位，范围±4KB，跳近处
伪直接
用于无条件跳转（jal），偏移量20位，范围±1MB，跳更远的函数
若函数超过1MB怎么办？用 JALR 指令（I型）：先 auipc 加载高20位到寄存器，再用寄存器+偏移量，组合出完整64位地址。
