01 寄存器速查 基础
寄存器名称用途考试映射惯例
$zero / $0零寄存器永远是 0，只读，硬件写死—
$s0–$s7保存寄存器对应 C 程序变量f→$s0, g→$s1, h→$s2, i→$s3, j→$s4
$t0–$t9临时寄存器存中间计算结果做题最常用！
$s6, $s7（习惯用法）数组基地址A基址→$s6, B基址→$s7
⚠️
$0 vs $s0 易混淆！ $0 永远是常数 0；$s0 是变量寄存器。题目字体模糊时一定要放大确认，两者答案天壤之别。
02 三种指令格式 重点
R 型 — 寄存器运算
op6b
rs5b
rt5b
rd5b
shamt5b
funct6b
指令：addsubsllsrljr
add $rd, $rs, $rt # rd=rs+rt，三个都是寄存器
I 型 — 含立即数

rs5b

J 型 — 跳转
op6b

指令：jjal
j target # 实际地址=PC高4位|target|00
03 核心指令速查 重点
指令格式语义注意事项
addadd $rd,$rs,$rtrd = rs + rt三个操作数必须全是寄存器
subsub $rd,$rs,$rtrd = rs - rt无 subi！减常数用 addi 加负数
addiaddi $rt,$rs,immrt = rs + imm立即数16位，符号扩展，范围±32767
oriori $rt,$rs,immrt = rs | imm零扩展，加载大常数低16位用它
lwlw $rt,offset($rs)rt = Memory[rs+offset]offset必须是常数；执行后$rt装的是数据
swsw $rt,offset($rs)Memory[rs+offset] = rtoffset必须是常数
luilui $rt,immrt高16位=imm，低16位=0加载大常数第一步
sllsll $rd,$rt,shamtrd = rt << shamt= ×2ⁿ；数组×4固定用sll 2
srlsrl $rd,$rt,shamtrd = rt >> shamt= ÷2ⁿ（逻辑右移，高位补0）
jj targetPC = PC[31:28]|target|00只能跳同一256MB区域
jrjr $rsPC = $rs可跳任意地址，无范围限制
04 高频踩坑点 必看
⚠️ 陷阱 1：没有 subi

MIPS 没有减立即数指令，减常数只能用加负数。

sub $t0, $s2, 5 # ❌ 不存在！ addi $t0, $s2, -5 # ✅ h-5 = h+(-5)
⚠️ 陷阱 2：add 不能用常数

add/sub 是 R 型指令，两个源操作数必须是寄存器。

add $t0, $s0, 5 # ❌ 5不是寄存器 addi $t0, $s0, 5 # ✅ 用addi
⚠️ 陷阱 3：数组下标要×4

int 占4字节，变量下标必须先转换为字节偏移。

# A[i] 的地址 sll $t0, $s3, 2 # t0 = i×4 add $t0, $s6, $t0 # t0 = &A[i]
⚠️ 陷阱 4：lw 后寄存器身份变了

lw 执行后，目标寄存器装的是数据，不再是地址。

add $t0, $s6, $t1 # $t0 = 地址 lw $t0, 0($t0) # $t0 = 数据（身份转换！）
⚠️ 陷阱 5：逆向读汇编追踪寄存器

$s0 可能中途被覆盖，后面再用时已是新值，不是原始变量 f！

lw $s0, 0($t0) # $s0 从此变成 A[f] # 后面的 $s0 ≠ 原来的 f！
⚠️ 陷阱 6：addi vs ori 扩展方式不同

加载大常数低16位时必须用 ori，否则低16位最高位为1时会破坏高位。

# 低16位最高位=1（如0x9xxx）时： addi # 符号扩展 → 高16位变0xFFFF 💥 ori # 零扩展 → 高16位不变 ✅
05 数组访问完整流程 重点

以 B[8] = A[i-j] 为例（A基址$s6，B基址$s7，i=$s3，j=$s4）

# 步骤1：算变量下标 sub $t0, $s3, $s4 # t0 = i-j # 步骤2：转字节偏移（×4） sll $t0, $t0, 2 # t0 = (i-j)×4 # 步骤3：算地址，读值 add $t1, $s6, $t0 # t1 = &A[i-j] lw $t2, 0($t1) # t2 = A[i-j] # 步骤4：常数下标直接×4写offset sw $t2, 32($s7) # B[8]=A[i-j]，32=8×4
💡
优化技巧：addi+lw 可合并
看到 addi $t2, $t0, 4 紧跟 lw $x, 0($t2)，直接合并为 lw $x, 4($t0)，省一条指令。
💡
常数下标不需要 sll
B[8] 的偏移 = 8×4 = 32，编译期已知，直接写进 sw 的 offset 字段，不需要计算。
先 sll 再 addi 更优
# 访问 B[i+1] 和 B[i+2] sll $t0, $s0, 2 # t0=i×4 addi $t1, $t0, 4 # (i+1)×4=i×4+4 addi $t2, $t0, 8 # (i+2)×4=i×4+8 # 比先addi再sll少1条！
06 32位大常数加载 重点

以 0x20014924 为例

# 固定套路：lui + ori lui $t1, 0x2001 # $t1 = 0x20010000（低16位补0） ori $t1, $t1, 0x4924 # $t1 = 0x20014924 ✅
💡
两个 $t1 完全合法，意思是"$t1 = $t1 | 立即数"，源和目标同一个寄存器在 MIPS 里很常见。
addi vs ori 扩展对比
低16位最高位ori结果addi结果
= 0（如0x4924）0x00004924 ✅0x00004924 ✅
= 1（如0x8924）0x00008924 ✅0xFFFF8924 💥

结论：永远用 ori，不管最高位是 0 还是 1，一律安全。

07 大端 vs 小端字节序 考点

以 0xabcdef12 从地址 0 开始存储为例

大端 Big-Endian — 高字节在低地址
地址 0 0xab 最高字节
地址 1 0xcd
地址 2 0xef
地址 3 0x12 最低字节
✅
顺序与我们书写数字完全一致，直觉友好。
MIPS 教材默认使用大端；TCP/IP 网络协议也是大端。
小端 Little-Endian — 低字节在低地址
地址 0 0x12 最低字节
地址 1 0xef
地址 2 0xcd
地址 3 0xab 最高字节
🔄
顺序与书写完全相反，像把字节"倒过来"存。
x86/x64（你的电脑）是小端。
🗝️
考试答题口诀： 大端 = 大头（高字节）在前（低地址）；小端 = 小头（低字节）在前（低地址）。小端就是把字节顺序整体翻转后写。
08 j 指令跳转范围判断 考点

实际跳转地址拼接方式：

PC[31:28]4位
指令中的26位target26位
补00
高4位来自PC 中间26位来自指令 末两位固定00

末两位固定00的原因：MIPS 指令4字节对齐，所有合法地址末两位必然是00，存储时省略这2位。

快速判断法（考试直接用）
PC 十六进制第1位目标地址第1位结论
00✅ j 可以跳
02❌ 不能，需要 jr
22✅ j 可以跳

十六进制第一个字符就是高4位，相同才能用 j，不同就用 jr。

💡
跨区域跳转用 jr：
lui $t0, 0x2001 ori $t0, $t0, 0x4924 jr $t0 # 跳到任意地址
⚠️
写 j 指令时直接写完整32位地址，汇编器自动提取中间26位。高4位必须和执行时PC高4位一致，否则汇编器报错——它会检查但不会帮你修正。
09 两类题型的答题套路 方法论
C → MIPS（正向翻译）
STEP 1确认变量→寄存器映射（f=$s0等）
STEP 2按优先级拆分子操作
STEP 3一个子操作→一条指令
STEP 4数组下标×4，常数下标直接写offset
STEP 5检查能否合并（addi+lw → lw offset）
MIPS → C（逆向翻译）
STEP 1建寄存器追踪表，逐行更新每个寄存器的值
STEP 2标注每个寄存器当前是"地址"还是"数据"
STEP 3特别注意变量寄存器（$s0等）是否被 lw 覆盖
STEP 4sw 的目标地址反推数组名和下标
STEP 5整理成完整 C 语句
10 一句话知识速记 速查
0x 前缀

只是"我是十六进制"的身份证，本身无数值含义。1位十六进制=4位二进制，可读性远超二进制。

sll n = ×2ⁿ

左移1位×2，左移2位×4。数组寻址×4固定用 sll 2，比 mul 快得多（单周期 vs 多周期）。

4字节对齐

合法指令地址末两位永远是00，即十六进制末位必须是 0/4/8/C。

j 的256MB限制

j 只能在当前PC所在的256MB区域内跳。跨区域必须用 jr + lui+ori 先加载目标地址。

lui+ori 固定套路

32位大常数永远用 lui 装高16位 + ori 填低16位，两条搞定，永远用 ori 不用 addi。

add/sub 只吃寄存器

两个源操作数必须全是寄存器。有常数就用 addi；减常数就是 addi 加负数；MIPS 无 subi。
